Version 1.0.0.0 of splicer (the little video/audio composition library that leverages DirectShow which I started a few years ago) is now available on Codeplex
here</a> this marks a milestone in stability, and probably the main "feature" of this release is 64bit support, something that's been bugging me for ages as I could only work on the project in a VM!

A quick list of changes since the last release are:

  • Now uses DirectShow.Net 2.0 (thanks to felix, a fellow NZ'r).</li>
  • RenderProgress event.</li>
  • Renderers are disposable.</li>
  • Support for 64bit operating systems.</li>
  • Vista fixes/support.</li>
  • Additional samples (i.e. SampleTimeWatermarkParticipant, and a few others).</li>
  • Tests updated for NUnit 2.4.7.</li>
  • Solution upgraded to VS2008.</li>
    </ul>

    What's splicer?</span></h2>
    With this library and a little imagination you can:

    • Encode video or audio suitable for use on a website.</span></li>
    • Create slide shows from images, videos and audio.</span></li>
    • Apply effects and transitions to audio and video.</span></li>
    • Grab System.Drawing.Image</strong> clips from a video at certain times.</span></li>
    • Modify individual video frames during encoding via standard C# mage Drawing code.</span></li>
    • Add new soundtracks to existing video clips.</span></li>
    • Watermark videos.</span></li>
    • Build a video editing suite, if you were so inclined.</span></li>
      </ul>
